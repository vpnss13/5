Shadowsocks软件介绍

SSR，全称为 ShadowsocksR（下文简称为SSR） 来历：

ShadowsocksR是breakwa11发起的Shadowsocks分支，在Shadowsocks的基础上增加了一些数据混淆方式，称修复了部分安全问题并可以提高QoS优先级。后来贡献者Librehat也为Shadowsocks补上了一些此类特性，甚至增加了类似Tor的可插拔传输层功能。

ShadowsocksR开始时曾有过违反GPL、发放二进制时不发放源码的争议，不过后来ShadowsocksR项目由breakwa11转为了与Shadowsocks相同的GPL、Apache许可证、MIT许可证等多重自由软件许可协议。

2017年7月19日，ShadowsocksR作者breakwa11在Telegram频道ShadowsocksR news里转发了深圳市启用SS协议检测并被大量用户转发，引发恐慌。7月24日，breakwa11发布了闭源的SS被动检测程序，引发争议。 2017年7月27日，breakwa11遭到自称 “ESU.TV” 的不明身份人士人身攻击，对方宣称如果不停止开发并阻止用户讨论此事件将发布更多包含个人隐私的资料，随后breakwa11表示遭到对方人肉搜索并公开个人资料的是无关人士，为了防止对方继续伤害无关人士，breakwa11将删除GitHub上的所有代码、解散相关交流群组，停止ShadowsocksR项目。但项目已被多人fork。并有人在其基础上继续发布新的版本，例如SSRR

Shadowrocket 模式与使用方法
Shadowrocket 模式

Shadowrocket 支持多种工作模式，以满足不同用户的需求。以下是一些常见的模式：

全局模式：所有网络流量都通过代理服务器。这种模式适合需要全程保护网络隐私的用户。
自动模式：根据访问的 URL 自动选择是否使用代理，适合大多数用户的日常使用。
分流模式：可以设置特定的网站或应用通过代理，而其他流量不经过。这种模式在访问某些特定网站时特别有效。
Shadowrocket 使用方法

使用 Shadowrocket 时，可以按照以下步骤进行配置：

下载与安装：在 App Store 中下载 Shadowrocket。
添加代理：打开应用，选择“配置”，点击“添加代理”，输入服务器地址、端口和协议类型（如 Shadowsocks、Vmess）。
设置模式：根据需求选择合适的工作模式（全局、自动或分流）。
启动代理：完成设置后，启用代理，确保流量正常通过。
SS 与 Shadowrocket
SS（Shadowsocks）是一种常见的代理协议，Shadowrocket 支持该协议，用户可以通过配置 SS 服务器在网络上进行加密通信。使用 SS 协议的优势在于其快速且低延迟，适合需要快速连接的场景。

Shadowrocket 延迟问题
在使用 Shadowrocket 时，延迟可能会影响上网体验。以下是一些常见的导致延迟的因素：

服务器位置：选择距离较近的服务器可以降低延迟。
网络状况：不稳定的网络连接会导致更高的延迟。
服务器负载：过于拥挤的服务器可能会影响连接速度。
